IT Documentation : Integration Patterns

Created by r.lodder1, last modified on mrt. 25, 2022

Scenarios

Cloud Pub to Internal/Cloud Sub

Example: Workday Hours to Internal IFS and ISAH

External facing adapter for Workday is created in the Events API w/ config file as Integration Layer

Events publishes on AnypointMQ Exchange

Optional: Goes to event handler for additional decisions

Subscribers defined in internal facing APIs e.g. Hours w/ config file as System Layer

Internal Pub to Internal/Cloud Sub

Example: IFS Hours to ISAH and Workday

Internal facing adapter for IFS Hours is created in e.g. Hours API w/ config file as Integration Layer

Event is published on AnypointMQ Exchange

Optional: Goes to event handler for additional decisions

Subscribers defined in internal facing APIs e.g. Hours w/ config file as System Layer

Internal Request/Reply APIs with specific mapping

Example: DMS performs sync HTTP call for retrieving IFS contacts data where DMS demands its data in its own format

Internal facing adapter for DMS specifically defined in IHC Integration API

Mule flow calls Process/System Layer API

Internal Request/Reply APIs without specific mapping

Example: DMS performs sync HTTP call for retrieving IFS contacts data where DMS demands its data in its own format

Internal facing adapter for DMS specifically defined in IHC Integration API

Mule flow calls Process/System Layer API

External Request/Reply APIs

Example: Mendix wants to retrieve customer information

Web App Experience Layer contains spec and implementation for customer specific and is outside facing

Web App calls internal process/system layer

Internal Scheduled tasks

Direct implementation in system layer
