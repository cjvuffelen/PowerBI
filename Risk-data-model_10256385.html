IT Documentation : Risk data model

Created by Martjan Pols, last modified by Chenxi Liang on jun. 03, 2022

The risk power bi data model consist of several tables from IFS having different grains, with the purpose to manage the cost and delays for the project. For every project one or more risk analyses can be created. For every risk analyses one or more risks can be identified, where the risk analyses can have multiple revisions. For every risk one or more responses can be added and for every response actions can be defined.

Sub project

1:n risk analysis

1:n risk analyses revision

1:n risks

1:1 risk problem

1:1 risk consequence

1:n risk response

1:n risk action

Technically also the risk, risk problem and consequences are 1:n, but this is locked in IFS.

Data model picture (power BI)

The data model in power bi has one periodic snapshot table with the grain of month end and a history of two years.

The fact is created using the lowest grain of the risk related tables: risk action.
