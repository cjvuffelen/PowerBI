Help and support settings

Option

Setting

Reasoning

Receive email notifications

Enabled for tenant admins

Pro-actively be aware of main incidents related to Power BI service. Only tenant admins should get notifications, and they decide to further inform.

Allow users to try power bi paid features

Disabled for organization

A pro license is automatically connected to the user once connected to the service. person will start building and needs license after 90 days. prefer to control the access by adding a person to an ad-group.

Power BI visual settings

Custom Visuals can interact with the data and are not prevented to send out data outside the service. therefore it is extremely important to handle custom visuals with care (read power bi white paper on enterprise deployments for more detailed information).

Documentation on settings: https://docs.microsoft.com/en-us/power-bi/admin/organizational-visuals

The access settings for custom visuals can be managed on tenant level and on pbi desktop level (using AD groups)

Tenant level

Option

Setting

Reasoning

Allow visuals created using Power BI SDK

Enabled

As IHC wants to use certified visuals, this needs to be enabled.

Add and use certified visuals only

Enabled

As there is a potential security risk with visuals not being certified, this option should be enabled to block not certified visuals.

Allow downloads from custom visuals

Disabled

As there is a potential security risk the custom visual is not allowed to download data.

Exceptions can be made using the organizational section in the admin portal.

Power BI Desktop level

With the use of AD group policies, IT can manage the download of not certified visuals from the market place. Advice is to limit the download and usage of not certified visuals by pro users, and allow it only for CDM power bi developers (Currently not implemented).

Organizational visuals

The organizational visual page can be used to by-pass the existing tenant settings, but there are more advantages.

Advantages:

  1. By-passing the existing tenant settings.
    Any visual deployed via power BI desktop which is in this list is automatically accepted by the service. So this is a work around for custom visuals which are not certified but still needed. (the exception). This is only maintained by the tenant admin. (please always use the development guidelines on custom visuals).

  2. With the option “enabled” on visualization pane, the custom visual will be added to every power bi desktop connected to the service. End users don’t need to add this custom visual as it is a company accepted custom visual.
