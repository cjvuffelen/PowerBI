Synoptic panel on section progress

Explanation of the Section progress visual with the synoptic panel.

Requirement

The requirement was to visualize the progress on sections using colors which identify how far the progress was. Using power BI as reporting tool, the only realistic option is to use custom visual Synoptic Panel.

Process workflow

The following process is based on conclusion of analyzing the possibilities of the synoptic panel (and other custom visuals) as documented here.

Before building the visual, the following should be available in the data:

  • Project (code, description) for a slicer

  • Section code, description for the tooltip

  • Technical combination of project code and section code stored as [project code]_[section code]. like E000579_1101, used for the connection with the visual. The combination will ensure only that section on that drawing otherwise the 1101 section will also get a color for other projects.

  • A measure who counts the finished activities in total over the phases.

Create svg file

After the JPG file is received it needs conversion to add data points. This can be done using the synoptic designer.

  1. Open the JPG file in designer

  2. Follow the steps as documented online.

  3. Add the section name using the naming convention as described

  4. Do this for all sections and follow the steps to create the svg file.

  5. Store this file here: ….\Github\PowerBI\Workspaces\IHC integrated project performance\Custom Visuals\Section progress\ using the name of the boat. The name of the file will be visible to end users and must therefore be meaningful to them

  6. Import all the drawings from the location using the toolbar in the custom visual

  7. Select all the svg files in the folder and import.

  8. Publish and deploy.

Synoptic panel on risk matrix

Documentation of the risk impact probability matrix

Requirement

Deliver an impact probability matrix based on IFS data. Matrix should be selectable using the default colors for a risk matrix (red / yellow / green) and consist of 5x5 cells.

Process flow

After searching for a solution on the internet, the custom visual Synoptic panel is selected. The process is designed based on this video here.

Before building the visual the following information should be available in the data:

  • Risks connected to a project including impact and probability

  • Dimension risk contains impact probability code column using numbering from 1 to 25

  • Colors used for red green and yellow:
    green: 299617

    red: e1001a

    yellow: ffdf46

Create matrix

All information related to the visual is stored in git: …\Github\PowerBI\Workspaces\IHC integrated project performance\Custom Visuals\Risk matrix\

  1. Open prepared excel file risk_matrix

  2. Make sure the naming of the impact column and probability column matches the names in IFS

  3. Validate the case when statement generated using excel, and adapt if needed in the risk dimension

  4. Create a picture of the matrix and store it in the location

  5. Make svg file in synoptic designer following the documentation online

  6. Connect the custom visual using the created impact probability code in the risk dimension

  7. Follow the video instructions to smooth the layout.
