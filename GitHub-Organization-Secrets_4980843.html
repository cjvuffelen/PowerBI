IT Documentation : GitHub Organization Secrets

Created by r.lodder1, last modified on jul. 07, 2022

GitHub Organization Secrets are secrets available for the complete organization and are perfect if certain secrets should be available. For the MuleSoft API Github workflow to work properly, different properties are used to actually execute the CI/CD tasks. A complete list of the organization secrets can be found in Github. However, the following “types” of organization secrets are defined.

  • Azure Tenant: Tenant ID for calling the correct Azure environment.

  • GitHub token: Token created for reaching the Github environment and checking out the code in the workflow. This token is created from the user ‘royalihcanypointadministrator’. It is important that this user is used for both checking and the code and reaching the Github Packages. Without this user, checking out code and merging code would fail. The token can be found when logging in with this user in GitHub.

  • Nexus credentials: Retrieving MuleSoft dependencies from MuleSofts' Nexus environment. These dependencies are components like MuleSoft Dataweave. This is mandatory to build a MuleSoft Enterprise application

  • MuleSoft CloudHub Client credentials: Contains the credentials of the MuleSoft Connected App (https://docs.mulesoft.com/access-management/connected-apps-overview ) which makes it possible to deploy applications and download artifacts from Anypoint Exchange. This ‘Connected App' is different than a normal Anypoint Platform account. This connected app only has specific access to certain components of the platform, such a Exchange and deployment to Anypoint Platform Runtime Manager. Access via the GUI is not possible.


  • Anypoint Platform Client credentials: Contains the credentials that are being set as properties for deployment. In this way, the correct credentials set is defined for the correct environment. In this way, the MuleSoft application can access e.g. API Manager. This is different than the MuleSoft CloudHub Client credentials. These credentials are not Anypoint Platform user credentials. If it purely a credentials set for the environment without a business group.
