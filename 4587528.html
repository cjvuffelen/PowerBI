IT Documentation : Setup CI/CD in GitHub Team

Created by r.lodder1, last modified on mrt. 15, 2022

GitHub Overall Settings

The MuleSoft Integration CI/CD is relying on settings defined on ‘Team level’ and ‘Repository level’. The ‘Team level’ settings apply for every repository in that team. The repository setup only applies to the repository. However, for Integration repositories this should always be the same.

Access to repositories

When creating a new GitHub repository, it is not automatically implied that a certain team has access to this repository. Only the repositories defined in the repositories overview in the Team are candidates for auto assignment.

Team level auto assign

The MuleSoft Integrations are being developed by the team members of ‘Integration’. This group is responsible and knowledgeable to create and maintain these integrations. Therefore, everyone in this group can be assigned in reviewing PRs without getting overloaded with requests.

To achieve this, the ‘Team settings’ must be changed for ‘Code Review’.

The following decisions where made for ‘Code Review’ on Team level:

  • Notification - Only the persons assigned to perform the PR will be notified

  • Auto Assign - When a PR is created, a person is automatically assigned to do the review

  • Team members to review - Only one person needs to perform the review

  • Routing Algorithm - Load balancing is used, meaning that GitHub takes into consideration how many review jobs are already on someones name before assigning it
