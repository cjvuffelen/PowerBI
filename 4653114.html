IT Documentation : Main CI/CD workflow

Created by r.lodder1, last modified on mrt. 17, 2022

Overview

The most important GitHub Workflow of all is the Main CI/CD Workflow. This reusable workflow is created to handle all the CI/CD related topics including deployments to the different environments of Anypoint Platform. This Main CI/CD Workflow is a reusable GitHub Workflow used by all Mule APIs to provide a consistent CI/CD experience.

This workflow is reflecting the “normal” CI/CD lifecycle. However, how it is implemented is always up to the CI/CD strategy. Points like ‘What should the test coverage be and who should test it?’, ‘How do we do released?’ and ‘Do we perform automated deployments to production?’ can always be different per company. The baseline however is always the same.

This workflow contains the following jobs to support the CI/CD lifecycle:

  • Build Job

  • DeployToDevTest Job

  • DeployToQa Job

  • DeployToProd Job
