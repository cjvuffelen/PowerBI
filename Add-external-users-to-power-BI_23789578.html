An external user is a user not in the active directory of IHC.

Pre-requisite

The following admin settings should be enabled for viewing only:

With these settings it is possible by admin group only to add external users. For more information regarding external users please read documentation here. The settings and why they are enabled is documented here.

Add user

Adding the user and limit access right is possible on multiple levels.

On app level

Additional rights can be set on app level after clicking the update app button in the workspace. This can be done in the permissions tab:

Add the external user like abcd.efg@gmail.com This will add an external azure user in the backend: abcd.efg_gmail.com#EXT#@ihcmerwede.onmicrosoft.com. This user is not in the local active directory, but only in azure, and will automatically be removed once the rights are revoked

Please reduce access rights further by checking the right box:

With unchecked, the external user is not able to re-share or build on the underlying dataset.

On workspace level

In cases the underlying dataset is in a separate workspace the external user needs to have viewer access to that workspace as well. This can be managed on workspace access.

Add the external user as viewer by looking up the abcd.efg which will result in the user: abcd.efg_gmail.com#EXT#@ihcmerwede.onmicrosoft.com
