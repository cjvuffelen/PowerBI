IT Documentation : DeployToDevTest Job

Created by r.lodder1, last modified on mrt. 17, 2022

Introduction

Relying on the Build Job, the DeployToDevTest job is an extensive job consisting of multiple steps. With the CI/CD lifecycle in mind, this jobs is at the end responsible to deploy to the Anypoint Platform DevTest environment. To get there however, the tasks build (code and retrieval of properties), test (Munit) and deploy of the CI/CD lifecycle must be executed successfully. These steps are already some security measurement to ensure that technically the code works as it should be.

Trigger, steps and variables

Trigger

This job is triggered in case of a Pull Requests from a ‘development branch’ to the ‘main branch’. The reason is that modified or added code must first be tested on the DevTest environment, then be approved in the PR before is can eventually be rolled out and tested on the QA environment.

The job is fully relying on the build job. This means that the https://ihcconfluence.atlassian.net/wiki/pages/resumedraft.action?draftId=4948164 always runs first and after that each job will look what kind of event it is (in this situation, its a PR).
