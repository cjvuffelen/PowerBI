IT Documentation : Create a new MuleSoft API foundation

Created by r.lodder1, last modified on jul. 05, 2022

This how-to article explains how to set up a new MuleSoft API based on Royal IHC Frameworks.

 Instructions

  1. Create a new repository from the repository template and change the repository settings (refer to article)

  2. Download the template from Exchange and import this template in Mule. Change the naming and pom file accordingly
    <<pom file>>

    1. artifactId → mule-<projectName>, e.g: mule-ciam

    2. version → Change version to 1.0.0-SNAPSHOT

    3. Remove the ‘configuration’ object in the ‘deploy’ execution object.

      <configuration>
	<classifier>mule-application</classifier>
</configuration>

    4. Change the classifier in the mule-maven-plugin from mule-application-template to mule-application

    5. Replace the complete distributionManagement object to the following object below. Replace <<projectname>> with your projectname. For example: https://maven.pkg.github.com/Royal-IHC-BV/mule-ciam

      <distributionManagement>
	<repository>
		<id>github</id>
		<name>GitHub OWNER Apache Maven Packages</name>
		<url>https://maven.pkg.github.com/Royal-IHC-BV/mule-<<projectname>></url>
		<releases>
			<enabled>true</enabled>
		</releases>
		<snapshots>
			<enabled>true</enabled>
		</snapshots>
	</repository>
</distributionManagement>

    6. Add repository object

    7. Change SCM from scm:git:https://github.com/Royal-IHC-BV/mule-api-temmplate.git to your correct Github repository name e.g scm:git:https://github.com/Royal-IHC-BV/mule-ciam.git

  3. Checkout the repository, create a new branch and add the just created Mule project to this branch
