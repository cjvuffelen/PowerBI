Introduction

The Build Job is a lightweight job that forms the basis of the DeployToDevTest Job and the DeployToQa Job. It sole purpose is to checkout the callers' repository. This code will then be used for all the upcoming tasks e.g. testing and compiling.

Trigger, steps, variables and config

The build job is triggered by any event and is a basis for other jobs. The build job consists of one step and no environment variables. This job runs on ubuntu-latest, which is the preferred environment of GitHub.

The checkout step is a GitHub Action that checks out the code from the calling repository. By default, the default branch (as set in the repository) is checked out. Once checked out, the job is complete
