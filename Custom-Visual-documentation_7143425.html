Security considerations

As there is a security risk, IHC have decided (in close collaboration with Security officer) to only allow certified visuals. (see premium settings). Third parties may send data out of the service. Power BI doesn’t prevent that. Certified visuals are tested by Microsoft.

Custom visual validation process

If the business request custom visual to be added to the service the following process needs to be followed:

Certified:

  1. Download from the market place.

  2. Download example pbix file (if applicable).

  3. Add to Github

  4. Add to organisational visuals from AppSource

  5. Enable on visualization pane

Un-certified

  1. Challenge the requester:

    1. Is there an alternative (certified)

    2. Is there business value

    3. Is it really needed

  2. If yes, prepare email to security officer with the request and details

  3. After confirmation download visual and example file

  4. add to Github

  5. add the visual to organisational pane by adding from file! not from app source as this will open automatic updates on this visual:

  6. Do not enable the visual on the visualisation pane

Usage and findings

The visual is tested using version 1.5.0.0 on march 2022 by Martjan and Patricia.

The custom visual is able to create a visualization in any shape we want with a status overview. this is very powerful in case we want to visualize for example the progress on sections in a boat. So, by selecting the project you should be able to see the corresponding boat as visual appearing with the status on the sections.

To have this dynamically changing on project selection, the visual needs to be connected to the right project in the data. There are three ways to achieve this.

  1. Import / embed multiple visuals

    1. In power bi desktop click on change in the visual and load multiple svg files at once.

    2. After changing the project in for example a slicer, the end user needs to manually also change the drawing in the custom visual

  2. Embed multiple visuals in the data

    1. store the files locally with a max of 32kb (zipped) file (locally in Git)

    2. Add the content of the file as described in here: https://okviz.com/blog/introducing-synoptic-panel-v1-4/

    3. Link that data to the project dimension

    4. Connect the added attribute to the visual (map)

    5. Now after the change of the project the selected drawing is also changed

  3. link data to online drawing

    1. Put the svg files on a public Git repo

    2. Add a link to the repository as attribute in the data structure next to the projects

    3. Link that attribute to the visual (map)

    4. After changing the project the drawing will change too.

Pro’s in general on the usage of the visual:

  • Enough documentation to use in production see links above

  • works very intuitive

  • many options to saturate or color objects

Cons on visual usage

  • larger files (more than 32kb) cannot be embedded locally (option 2 is not possible)

  • Only storage of files is only possible in public Github, so no use case for sensitive drawings (option 3 is not possible for sensitive drawings)

Final conclusion:

For sensitive drawings like a boat, which are greater then 32kb zipped each, only option 1 is working.
