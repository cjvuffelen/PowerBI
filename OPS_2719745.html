Adding or deploying new source

When adding or deploying an new source to a new invironment the copy activity fails:

'AuthorizationPermissionMismatch'. Message: 'This request is not authorized to perform this operation using this permission.'.

This means the rights are not set or set correctly goto github actions:

https://github.com/Royal-IHC-BV/IaC/actions/workflows/add-new-folder-acc.yml

and run this action for the relevant environments. Effect is directly.

DBT:

Running Silver layer and Model layer in parallel can cause an overload on Databricks.

The dbt log contains:

Action: is to restart DBT → dbt Silver and dbt model.

Due to the sudden crash a model table can be unusable showing the next error in the log:

Action: Drop the table from the model database, clean the stored files for that particular table (dbfs:/user/hive/warehouse/model.db/<table> and then restart the DBT run.
