IT Documentation : Components API

Created by Chi Chuen Ng, last modified on mrt. 31, 2022

The components API retrieves data from IFS and returns it to the client synchronously. It is part of the MARS-DMS interface and DMS uses it to get components, rooms and sections. Please click here for more information about the Components System API spec.

 

Components

The API gets components by calling stored procedure IFSAPP.IHC_PORTAL_UTIL_API.Get_Components(:cv_1,:last_modified_,:ihc_project_id_,:object_id_) where cv_1 is a output cursor and the rest are input variables. The last_modified_ variable expects a datetime format of “yyyyMMddHHmmss“. All variables are optional, however, the Components System API spec requirements are different.

The API spec expects projectId, lastModifiedDateTime (format: “yyyy-MM-dd’T’HH:mm:ss’Z'“) and objectId queryParameters of which the first two are required. ObjectId is an optional parameter to filter the collection for a specific component.

The flow itself is simple: a stored procedure call is made and the output is transformed into an JSON response.
