IT Documentation : IHC Integrations API

Created by Chi Chuen Ng, last modified on mrt. 31, 2022

The IHC Integrations API is an integration layer application. It will house many different integration or adapter APIs ranging from internal facing, external facing, synchronous and asynchronous APIs.

 

DMS Integration API

This internal facing, synchronous API serves as an adapter for DMS calling Contacts API and Components API. It retrieves contacts, contact addresses, components, rooms and sections data from those APIs and transforms them into JSON message formats for DMS.

Contacts

The adapter retrieves contacts by making a HTTP request call to Contacts API’s get:/contacts resource. The API spec expects projectId, lastModifiedDateTime (format: “yyyy-MM-dd’T’HH:mm:ss’Z'“) and contactId queryParameters of which the first two are required. Contactid is an optional parameter to filter the collection for a specific contact.

The flow itself is simple: a http request is made and the output is transformed into an JSON response for DMS.
