IT Documentation : Deployment review checklist

Created by ma.vanommering, last modified on jul. 08, 2022

IngestionConfig
Check the KeyColumn in the IngestionConfig (in the deployment file).
Although it is not absolutely necessary (only in case of delta load) it is better to have the right fields mentioned in the KeyColumn.
Compare them to the unique key in the staged sql.

Raw

Staged

- should contain the following filter: WHERE dt = ‘{{ var("dt") }}'
VRAAG FABIAN Unless in IngestionConfig the Snapshot = 'Insert’, then no filter on date.
- check the unique key in the staged sql if really unique via checking in SQL.
Example: select doc_class || '^' || doc_no || '^' || doc_sheet || '^' || key_ref || '^' || lu_name from doc_reference_object_tab
group by doc_class || '^' || doc_no || '^' || doc_sheet || '^' || key_ref || '^' || lu_name having count(*) <> 1

Silver layer

Model layer
- check if the silver layer view is used
- review the check in the yml file of the model layer (e.g. mdl_fact_risk_actions.yml) dbt_expectations.expect_compound_columns_to_be_unique.
If this concerns fields which are in the hsk field check if they are unique.
Example SQL to check:
select risk_id||risk_action_id, count() from datawarehouse.dwh_fact_risk_actions
group by risk_id||risk_action_id
having count() > 1
