Introduction theme development

Themes are a powerful way to control the layout or your report. for an introduction to themes in power bi click here. The purpose of a theme is to align the company report layouts in the same way using one pre-defined theme. A theme is used to control almost any part of the visuals used in power bi.

Important note:

A theme change will only influence the default settings of the visual on the report. If you add a visual in power bi, and change the default color for that specific visual, the color will stay even on a theme change. Therefore try to use default colors and settings in your power bi report.

Understanding the theme structure.

A theme is a json structure which inherit the settings from the highest level all the way down to the lowest level. and looks like:

Once a visual style is set, all visuals having the same setting for style will inherit the setting. For example the above picture shows a solid border with color f2f2f2, all visuals having a border will have a solid border with that color even if it is not mentioned in the theme json. If you want to differentiate a visual using a specific border color you need to include that visuals json structure overwriting the default border color. Please take a look at json structures here.
